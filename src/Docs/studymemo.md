next.js のメモ
ネクストはファイルの規則によって自動でルーティングを構築してくれる
サーバーでレンダリングできる
直感的なルーティングシステム（APP ルーターのほうが推奨）
ページごとにレンダリング戦力をとれる
Staticrendaring,dynamicrendaring がアップルータにはある
.next がネクスト js を動かすためのファイル
子コンポーネントに渡すときプロップスっていうネーミングにしがちだけど底別に何でもいい。
親コンポーネントになんて書くかが大事
⇒ 親のプロップスをいじってから子のプロップスをいじる
props には配列や数値、コンポーネント・JSX も渡せる
基本的に親からデータを渡して表示とかメソッドを出しわける
チルドレンを使うときは普段の閉じタグみたいに二つコンポーネントのタグを作って、その中に記述する
真ん中に入れたときはチルドレンという名前で、親コンポーネントのデータが子に移動する。名前をつけないときに使うだけ
cssmodules は css の命名をかぶせないようにするためのモジュール
